# Security Policy

## 报告漏洞

请通过代码托管平台的私密安全报告功能联系维护者。不要在公开 Issue 中发布邀请文件、访问码、私钥、服务器日志、设备 ID 或可直接利用的细节。

## 安全边界

- 织梭的固定服务器模式使用共享访问码鉴权，不是多租户权限系统。
- 公网固定服务器必须使用 HTTPS，并由部署者负责系统补丁、防火墙、备份和访问码轮换。
- 局域网本机托管模式默认使用 HTTP，只应运行在可信局域网或可信 VPN。
- 设备互联模式的 `.tdjoin` 文件包含限时加入秘密。应通过可信渠道发送，完成配对后删除；陌生设备仍必须由现有成员明确批准。
- 设备互联使用 Syncthing 的 TLS 设备连接。公共发现服务和中继可能看到设备 IP、设备 ID、连接时间和流量大小等元数据，但中继不能解密传输内容。
- 设备互联的内容以普通文件形式保存在每个成员设备的当前用户数据目录。磁盘加密、Windows 账户安全和设备处置由用户负责。
- 应用到期清理不保证从文件系统日志、磁盘快照、备份或第三方恢复工具中不可恢复。
- 公共发现和公共中继是外部服务，本项目不承诺其可用性或服务等级。

## 开源构建

Syncthing 通过 `tools/fetch-syncthing.ps1` 下载固定版本并校验 SHA-256。发布构建不应包含真实的 `deployment-config.json`、服务器证书、访问码或测试数据。
